The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- No-scroll full-screen shell (feature `5`): the site is now a fixed, single-viewport interface that never scrolls, inspired by `pacomepertant.com`. Content lives in full-screen *states* (Home / Work / About / Contact) swapped via a menu rather than stacked down a scrolling page. A `ShellProvider` (`src/components/shell/`) holds the state machine (active view + menu open/close); `ShellStage` crossfades between states; only the Home (hero) state is built — `work`/`about`/`contact` render a noise-register `PlaceholderState` until their states land in later features. The existing `Work`/`About`/`Contact` section components are parked (kept, no longer rendered) for reuse as those states.
- Minimal corner-marks navigation (`ShellNav.tsx`): a temporary neutral wordmark top-left (returns to Home), the language + theme toggles and a `Menu`/`Close` button top-right, and a full-screen menu overlay (`ShellMenu.tsx`) listing the states as a large editorial serif list. The bar is non-interactive except its marks, so it never intercepts clicks meant for the state beneath it. The menu manages focus (moves into the list on open, returns to the toggle on close) and closes on `Escape`; its active/hover destination takes the single signal accent, which is allowed because the overlay owns the whole viewport (scarcity holds per-view). New shell CSS component classes (`.shell-wordmark`, `.shell-control`, `.shell-menu-link`) plus `nav.home`/`nav.menu`/`nav.close` and a `shell.inDevelopment` label across EN/ES/FR.
- Signal / Noise hero (feature `4`): the pointer-field engine now carries the headline. The `<h1>` is real DOM text sitting in the noise register (`--lab-noise-4`, always legible, indexable, first to paint); a canvas layer above it rasterizes the same text to a coverage mask (`HeroField.tsx`) so the fragments inside the letterforms resolve into a signal-colored fill under the pointer, sharpening the word out of scatter and letting it drain back behind the cursor. The word is never *revealed* by the field — DOM owns legibility, canvas owns resolution, and they are registered against each other. Coarse pointers and `prefers-reduced-motion` render the whole word resolved as one static frame with no loop, so a phone visitor still sees the finished headline; the field never blocks scrolling. Left-aligned editorial composition, new copy ("Noise becomes structure."), and a ghost CTA so the resolved headline is the single signal accent in the view.
- Glyph field in the engine (`src/lib/pointer-field.ts`): a `GlyphMask` type with a pure coverage sampler, a second finer fragment population (`glyph*` tuning) laid only over the headline ink and tagged `inGlyph`, and a `resolveWordStatically` helper for the static frame. Ambient fragments now cap at the brightest noise step and never take the accent, so the assembled word is the only accent-colored thing on screen.
- Pointer-field prototype (feature `3`): the Signal / Noise signature interaction, built as a Canvas 2D engine (`src/lib/pointer-field.ts`) plus a thin client wrapper. A field of short line segments rests in noise; the pointer creates a temporary radius of local order in which fragments rotate into alignment, lengthen, and take the `--lab-signal` accent, then decay back roughly three times slower than they resolved so the trailing edge lingers. Fragments never translate — orientation, length, width, and color carry the whole effect, which is what keeps it from reading as a particle trail. Colors are read from the CSS custom properties at runtime, so both themes work with no per-theme tuning. Frame-rate independent, DPR-aware, and the loop idles to zero scheduled frames once the field settles. `prefers-reduced-motion` renders one composed static frame and never starts the loop. All tuning lives in a single documented `DEFAULT_TUNING` object; the throwaway tuning page it was found on was removed when Feature 4 landed.
- Signal / Noise design foundation (feature `2`): a palette where color carries the concept — a near-neutral base, a four-step noise ramp (`--lab-noise-1` … `--lab-noise-4`) for unresolved states, and exactly one `--lab-signal` accent reserved for resolved states (alignment, focus, active, CTA). Every token defined in both themes at introduction, no deferred theme pass.
- Named, fluid type scale (`display`, `h1`, `h2`, `h3`, `body`, `small`, `label`), each step carrying its own size, leading, and tracking as tokens. Added Instrument Serif via `next/font/google` as the display face (`h1`/`h2`/`.lab-display`), with Geist Sans on `h3`/prose/UI and Geist Mono promoted to a deliberate label role (`.lab-label`, eyebrows, kickers, tokens).
- Shared motion vocabulary: duration tokens (`fast`, `base`, `slow`, `field`) and easing tokens, principally the signature `resolve` curve — quick to leave, long to settle. Exposed as CSS custom properties and as a typed module (`src/lib/motion.ts`) so CSS and the `motion` library share one source of truth.

- Holographic HUD panel system for the Process section (feature 47): a reusable `ProcessHudPanel` (`src/components/sections/process/ProcessHudPanel.tsx`) rendered as a DOM/CSS glass overlay layered over the WebGL canvas — thin glowing token-driven border frame, corner ticks, mono/technical type, step + title header, translated status indicator, metric rows, and a small deterministic SVG sparkline. Extended `src/data/process.ts` with an illustrative per-stage `hud.sparkline` waveform and added matching `process.hud` copy (shared labels + per-stage status/metrics, illustrative values only) to `en.ts`/`es.ts`/`fr.ts`. Unified the existing per-stage copy crossfade in `ProcessScrollRig` and the mobile `MobileProcessStage` onto the panel; all accents flow from the per-stage `--stage-signal` cyan/teal token and the scan shimmer self-disables under `prefers-reduced-motion`.
- Premium 3D rendering foundation for the Process section (feature 45): a shared material toolkit (`src/components/sections/process/processMaterials.tsx`) exporting transmission `GlassMaterial`, `BrushedMetalMaterial`, and `EmissiveCoreMaterial`, plus a fetch-free generated studio `ProcessEnvironment` (static-export safe — no HDRI URL), a `ProcessBench` (`MeshReflectorMaterial`), and a restrained bloom+vignette `ProcessEffects` composer. Added `@react-three/postprocessing`.
- Shared quality-tier system (`src/lib/useProcessQualityTier.ts`): a mobile-first `matchMedia` hook (mirroring `useIsDesktopViewport`) whose knobs (bloom, transmission samples/resolution, reflector/environment resolution, dpr) drive both the desktop `ProcessScene` and `MobileProcessScene` from the same primitives — mobile scales down (cheap glass approximation, low-res env/reflector, no composer) rather than waiting for a later catch-up pass.
- Initial Lamadrid Labs project context.
- Initial website direction for a simple, clean digital laboratory experience.
- Initial showcase scope for RicardoOS and Marina Cuesta.
- Lab design tokens (color, radius, shadow) and base typography wired into Tailwind v4.
- Homepage `Hero` section (`src/components/sections/Hero.tsx`) with kicker badge, gradient headline, and CTAs.
- Light/dark theme toggle in the `Nav` (`src/components/ui/ThemeToggle.tsx`): dark stays the default, with a deliberately retuned light token set. Defaults to OS preference, persists the explicit choice, applies the theme pre-hydration to avoid a flash, and respects `prefers-reduced-motion`.
- EN · FR · ES language toggle in the `Nav` (`src/components/ui/LanguageToggle.tsx`) with typed local dictionaries (`src/data/i18n/{en,fr,es}.ts`). English is the default and runtime fallback; a shared `Dictionary` type forces every locale to provide the same keys. `LocaleProvider` supplies the active dictionary and syncs `document.title` / meta description; a pre-hydration script sets `<html lang>` before paint. The choice persists to `localStorage` and honors `navigator.language` on first visit. Nav, Hero, and Footer now source all copy from the dictionaries.
- Root metadata in `src/app/layout.tsx`: title/description, `metadataBase` (`https://lamadridlabs.com`), Open Graph and Twitter/X card fields using `public/og-image.png`, and a `viewport.themeColor` of `#020812`. Wired the Lamadrid Labs icon pack (favicons, apple touch icon, `site.webmanifest`) already placed in `public/`.
- Static export configuration in `next.config.ts` (`output: "export"`, `trailingSlash: true`, `images.unoptimized: true`) so `npm run build` produces a static `out/` directory deployable to Hostinger shared hosting.
- `deploy/.htaccess` (version-controlled) for the Hostinger export: custom 404, forced HTTPS + apex canonicalization, `.webmanifest` MIME type, and long-lived immutable caching for hashed `_next/static/` assets.
- `.github/workflows/ci.yml`: runs `npm run lint` and `npm run build` on pull requests into `main`.
- `.github/workflows/deploy.yml`: on push to `main`, builds the static export and FTPS-uploads `out/` to Hostinger via `SamKirkland/FTP-Deploy-Action@v4.4.0`, wrapped in `Wandalen/wretry.action` for retry-with-backoff and using a commit-SHA-scoped `state-name` to avoid stale delta-delete failures.
- `DEPLOY.md` documenting hosting facts, the automated CI/deploy flow, a manual zip-upload fallback, and a curl snippet to verify a live deploy.

### Changed

- Converted the app shell to the no-scroll interface (feature `5`): `layout.tsx`'s body is now `h-svh overflow-hidden` with a `ShellProvider` + `ShellNav` + a single full-viewport `main`, and `page.tsx` renders only `ShellStage` instead of a stack of scrolling sections. Retired the fixed floating-pill `Nav` and the IntersectionObserver scroll-spy it depended on (there is no scroll to observe); active state is now the shell's current view. `navigation.ts` moved from href anchors (`navLinks`) to typed view keys (`navItems` / `ViewKey`).
- Rebuilt `Hero.tsx` for Signal / Noise (feature `4`): retired the centered lab composition with its gradient-blob glow and the old `Ideas enter. / Software leaves.` copy for a left-aligned editorial layout driven by the pointer field. Hero copy in the dictionaries changed shape — the `titleFirst`/`titleHighlight`/`titleAfter`/`titleBefore` split became a `titleLines` array (one entry per display line) — across EN/ES/FR, and the eyebrow lost its signal dot so the resolved headline is the only accent.
- Renamed `--lab-accent*` → `--lab-signal*` throughout so the token name states its job, and re-pointed all consuming components (`Hero`, `Work`, `Contact`, `Nav`, `Footer`, `ThemeToggle`, `LanguageToggle`).
- Wired the existing motion call sites (`MotionPrimitives`, `Nav`'s mobile menu) to the shared vocabulary, replacing three hardcoded easing arrays and four ad-hoc durations.
- `viewport.themeColor` updated to the new base background (`#0a0b0d`).
- Rebuilt the Stage 1 Reagent Selection rig (feature 48, `src/components/sections/process/ReagentRig.tsx`) to reference fidelity on the feature-45 foundation: a glass stabilization column (transmission `GlassMaterial`) as the processing chamber, a branded machine pedestal with a control-panel readout hint, a "reagent-lock" cage (emissive rings + struts) around the column base, a dark brushed-metal machine head, internal cyan energy the orb settles into, and a floating periodic-tile reagent motif — replacing the old low-poly tile-rack/vial-tray primitives. Extracted the pedestal into a shared `RigMachineBase` primitive (`rigPrimitives.tsx`) so the later stage rebuilds (49–52) reuse the same branded base. The stage-1 HUD (STEP 01 · REAGENT SELECTION, EN/ES/FR) is unchanged from feature 47.
- Reconciled the Process 3D palette (`src/lib/narrativeSignals.ts`) from 5 saturated signal hues (cyan/violet/amber/rose/teal) down to a disciplined monochrome cyan/teal + white family — per-stage `signal` tokens now read as subtle accent shifts within that family rather than competing colors, matching the approved reference renders. Applied the new foundation to the desktop and mobile Process scenes: themed scene background (so the desktop composer can't force opaque black), env reflections on existing metal, a premium glass Synthesis flask, and a reflective bench.
- Reworked the visual direction from a light "laboratory white" theme to a dark "lab at night" theme (deep navy background, glass panels, teal + violet signal colors, grid overlay), inspired by the `LLprototype3` prototype.
- Restyled `Nav` into a floating glass pill; restyled `Footer` onto the new dark tokens.
- Refined homepage visual detail with shared section/header/eyebrow/card surface utilities, lighter card blur/shadow treatment, normalized letter spacing, and tighter section rhythm.
- Documented the Feature 17 decision to skip a dedicated `/work` route while the site only has two projects that fit comfortably on the homepage.
- Documented the Feature 18 decision to skip project case-study/detail pages until real long-form project content exists.
- Replaced the placeholder `public/og-image.png` with a branded 1200×630 social preview matching the shipped dark theme (icon-pack badge, wordmark, tagline).

### Fixed

- Scoped TypeScript and ESLint checks to the app source so reference files under `context/prototypes/` no longer break `npm run build` and `npm run lint`.
- Improved light-theme contrast for accent-filled controls by using a dedicated accent text token.
- Hardened homepage mobile wrapping for long headings, project summaries, service copy, and tag chips so the 390px layout avoids horizontal overflow.

### Removed

- The floating-pill `Nav` (`src/components/layout/Nav.tsx`) and its usage (feature `5`), superseded by the corner-marks `ShellNav`. The `Footer` and the page's bottom register are gone from the no-scroll shell — `Footer.tsx` / `social.ts` are parked for a future Contact state, not deleted.
- The throwaway pointer-field prototype (feature `4` cleanup): `/prototype/pointer-field/`, `PointerField.tsx`, `PointerFieldLab.tsx`, and the `/prototype/` disallow in `robots.ts`. The engine (`src/lib/pointer-field.ts`) and its findings survive; the tuning lab does not.
- `--lab-accent-secondary` and the `.lab-gradient-text` two-accent headline treatment — a decorative second hue is off-concept under Signal / Noise, and the gradient read as the generic SaaS look the brand direction rules out.
- The chemistry-laboratory experience, in full, as the first step of the Signal / Noise redesign (feature 1). Deleted all 21 `src/components/sections/process/` components (the five stage rigs, the specimen/orb, the scene, table, bench, backdrop, HUD panel, scroll rig, and every mobile counterpart), their supporting libraries (`processLayout`, `useProcessQualityTier`, `narrativeSignals`), their data (`process.ts`), the `Process` section, and the `process` copy block from `en.ts`/`es.ts`/`fr.ts`. Also removed the lab-metaphor page decoration (`AmbientParticles`, plus the `ScrollProgress` pipe and its permanent `requestAnimationFrame` loop), the five `/prototype/*` dev-harness routes that were being published to production, and the dead `services.ts` / `notes.ts` data files (neither was imported anywhere, and `dict.services` never existed).
- Dependencies `three`, `@react-three/fiber`, `@react-three/drei`, `@react-three/postprocessing`, and `gsap`. This also resolves a latent break: `ProcessScrollRig` imported from `framer-motion`, which was never a declared dependency and resolved only via a hoisted transitive copy. Runtime dependencies are now `next`, `react`, `react-dom`, and `motion`. Total exported JavaScript dropped from ~4.1 MB to ~812 KB, and the three separate 860 KB chunks are gone.
- Removed `src/app/favicon.ico` in favor of the `public/` icon pack + `metadata.icons` (avoids a duplicate `/favicon.ico` route).
- Deleted unused Create Next App boilerplate SVGs from `public/` (`file.svg`, `globe.svg`, `next.svg`, `vercel.svg`, `window.svg`).
